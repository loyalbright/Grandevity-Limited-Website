### 浏览器游戏发行的隐藏复杂性

在现代发行一款 HTML5 游戏，绝不仅仅是把一个 ZIP 压缩档上传到伺服器那么简单。

各大浏览器游戏平台（Portals）都在不同的环境下运行，它们有着各自的：
* Iframe 嵌套环境
* 安全策略
* 广告 SDK
* 音讯播放限制
* 画面缩放要求
* 平台认证规则

在部署期间，确保游戏在所有这些平台上都能完美相容，成为了一个巨大的工程挑战。

### 解决 Iframe 缩放难题

最大的技术障碍之一，是如何在不可预测的浏览器环境中保持视觉一致性。当游戏被嵌入到第三方平台的布局中时，标准的响应式 CSS 缩放方法通常会导致基于 Canvas 的游戏画面变形。

为了解决这个问题，《Kiwi Defenders》使用了一个专用的 `resizeGame()` 缩放系统。它能动态计算长宽比，并应用受控的 CSS `transform` 缩放，同时完美保留原始的 1280×720 游戏解析度。

这确保了游戏画面在以下设备中保持清晰稳定：
* 超宽萤幕显示器
* 标准桌面显示器
* 平板电脑
* 行动端直式浏览器

### 遵守现代浏览器的可见性规则 (Visibility API)

大型发行平台对游戏的背景行为有着严格的要求。如果玩家最小化了浏览器视窗或切换了分页，游戏必须立即暂停、静音，并挂起活动渲染循环。

为了符合这些要求，游戏引擎透过监听 `visibilitychange` 事件，直接与浏览器的 Visibility API 整合。这不仅防止了背景播放音讯的尴尬，还有效降低了游戏处于非活动状态时不必要的 CPU 占用。

### 构建模组化的 SDK 整合层

不同的发行商使用截然不同的广告与平台 API。我们没有将商业化系统与核心游戏架构强耦合，而是围绕「模组化 SDK 整合层」设计了整个专案。

这种抽象化设计允许游戏快速在以下系统间无缝切换：
* 基于 Promise 的广告系统
* 全域事件监听器 (Global event listeners)
* 奖励式广告 API (Rewarded ad APIs)
* 发行商特定的储存框架

这一切都不需要重写底层的核心游戏逻辑。

### 面向长期可扩展性的工程设计

《Kiwi Defenders》的长期目标之一始终是「高移植性」。透过保持架构的模组化、轻量级与平台无关性，该专案能轻松适应未来新增的发行商整合、新的商业化变现系统、多语言部署以及长期的营运服务 (Live-service) 更新。

随着浏览器游戏生态系在桌面与行动端的持续演进，这种底层的灵活性正变得越来越具有战略价值。
